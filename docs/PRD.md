# Tal Travel - Product Requirements Document

## חזון הפרויקט

**Tal Travel** היא פלטפורמת תכנון וניהול טיולים אישית. מתחילה כפלטפורמה משפחתית של רוי ולאהי טל, עם אפשרות הרחבה למשפחות וחברים שמטיילים יחד.

המוצר ממוקד באנשים שמשקיעים זמן רב בתכנון טיולים ורוצים פלטפורמה אחת שמלווה אותם מהרגע הראשון של "אולי נטוס ליפן" ועד החזרה הביתה (ועד הטיול הבא).

**גורם הצלחה מרכזי:** הפלטפורמה צריכה להיות מספיק שימושית כדי שמשתמשים יבחרו בה על פני Google Docs / Notion / WhatsApp / TripIt.

- אוהב לתכנן בפירוט

- רוצה לראות את "התמונה הגדולה" אבל גם ירידה לפרטים
- מקפיד על ארגון מסודר ועדכון מתמיד

- מעדיפה ממשק פשוט ונקי

- מצטרף לטיול משפחתי/חברתי

## מטרות השימוש

הפלטפורמה משרתת שלוש פאזות של טיול:

1. **תכנון מקדים (Desktop primary)** - מהתכנון הראשוני ועד היציאה
   - בחירת יעד, תאריכים, מבנה
   - מחקר ובחירת מלונות, טיסות, אטרקציות
   - ניהול משימות ובוקינגס

2. **שימוש בזמן הטיול (Mobile primary)** - כל יום ביעד
   - דף "היום" עם מה צריך לעשות עכשיו
   - גישה מהירה למסמכים, אישורים, כתובות
   - עבודה offline למחסור באינטרנט
   - שיתוף מיקום, סטטוס בזמן אמת

3. **תיעוד לאחר הטיול (Hybrid)** - אחרי החזרה
   - יומן, תמונות, תובנות
   - "מה עבד" - בסיס לטיולים הבאים
   - שיתוף עם משפחה וחברים

## גישה Hybrid: דסקטופ + מובייל

זה לא Mobile-Only ולא Desktop-Only. כל חוויה אופטימלית למקרה השימוש שלה:

### Desktop Experience
- מסכים רחבים עם הרבה מידע בו-זמנית
- טבלאות, השוואות, ניהול ויזואלי
- ידידותי לקלדנות מהירה ועריכה
- אופטימלי לפאזת התכנון
- חלוקת מסך: רשימה משמאל + פרטים מימין (Master-Detail)
- מתאים לשעות התכנון בלילה במחשב

### Mobile Experience (PWA)
- ניווט בלשוניות תחתונות (Bottom Tabs)
- מסך אחד = משימה אחת
- כפתור FAB עגול לפעולות מהירות
- אופטימלי לשימוש ביד אחת
- עובד offline
- ניתן להוסיף למסך הבית
- מתאים לשימוש בטיול עצמו

### חוויה מסונכרנת
- אותם נתונים, אותם משתמשים, אותו בסיס נתונים
- שינוי מהמובייל מתעדכן מיד בדסקטופ (Realtime)
- התקדמות נשמרת בין מכשירים

## ארכיטקטורת המוצר

### היררכיה

```
Tal Travel Platform
└── User Account
    └── Trips (each user has many trips)
        ├── Members (who has access)
        ├── Modules (which features active)
        ├── Bases (locations within trip)
        ├── Daily Plan (calendar)
        └── Module Data
            ├── Hotels
            ├── Flights
            ├── Attractions
            ├── Tasks
            ├── Documents
            ├── Expenses (V2)
            ├── Journal (V2)
            └── Photos (V2)
```

### מודולים גמישים

כל טיול מפעיל רק את המודולים הרלוונטיים. דוגמאות:

| מודול | טיול משפחתי ארוך | סוף שבוע זוגי | טיול חברים |
|------|------------------|----------------|-------------|
| Hotels | ✅ הרבה | ✅ אחד | ✅ |
| Flights | ✅ | ✅ | ✅ |
| Daily Plan | ✅ מפורט | ⚠️ קל | ✅ |
| Attractions | ✅ הרבה | ⚠️ מעטות | ✅ |
| Documents | ✅ | ✅ | ✅ |

## פיצ'רים מרכזיים

### MVP (לפני יציאה ליפן ב-20.7.2026)

המטרה: פלטפורמה פונקציונלית לכל פאזות הטיול הראשון.

**1. ניהול משתמשים וטיולים**
- הרשמה והתחברות (Magic Link + Email/Password)
- פרופיל משתמש עם תמונה
- יצירת טיול חדש עם פרטים בסיסיים
- רשימת כל הטיולים שלי (Upcoming / Past)
- הזמנת משתתפים לטיול עם הרשאות
- מערכת הרשאות: Owner, Editor, Viewer

**2. דשבורד "Trips" (מסך ראשי)**
- כרטיסים יפים לכל טיול (כמו TripIt)
- כותרת, תאריכים, תמונת קאבר
- חלוקה: Upcoming / Past
- כפתור + להוספת טיול חדש

**3. דשבורד "Today" (היום) ⭐ קריטי לשימוש בטיול**
- מסך פתיחה ברירת מחדל בזמן טיול פעיל
- "Today's Plan" - ציר זמן של היום הנוכחי
- "Up Next" - מה הדבר הבא (טיסה בעוד 3 שעות?)
- "Documents I Need Today" - גישה מהירה לאישורים
- "Weather Now" - מזג אוויר במקום הנוכחי

**4. מסך פרטי טיול**
- Header: שם הטיול, תאריכים, משתתפים, countdown
- ניווט פנימי בין מודולים פעילים
- ציר זמן ראשי של כל הטיול
- כפתור FAB להוספת פריט

**5. מודול מלונות**
- כרטיסים מודרניים עם תמונות
- שדות: שם, מיקום, תאריכים, סטטוס, מחיר, אישור
- סטטוסים צבעוניים: לחקור / לבחור / להזמין / הוזמן
- חיבור למסמכים (אישור הזמנה)
- מסנן לפי בסיס/אזור

**6. מודול טיסות**
- כרטיסים בסגנון boarding pass
- שדות: חברה, מספר, מסלול, זמנים, סטטוס
- חלוקה: בינלאומיות / פנים / חיבורים
- חיבור למסמכים (כרטיסים, אישורי שיבוץ)
- הוספה אוטומטית ל-Daily Plan

**7. מודול לוח זמנים יומי**
- תצוגות: יומן (חודש), רשימה (יום), ציר זמן
- כל יום עם: בסיס, פעילויות, מלון, מזג אוויר
- תגיות צבעוניות: הגעה, מעבר, יום מנוחה, אטרקציה
- שילוב אוטומטי של טיסות, אטרקציות עם זמנים

**8. מודול אטרקציות**
- רשימה ארוכה עם סינון
- סטטוסים: מעוניין, מתוכנן, בוצע, דילגנו
- מיון: לפי בסיס, קטגוריה, חובה-רצוי-אפשרי
- אפשרות "לקבוע" - מעביר ל-Daily Plan

**9. מודול משימות**
- חלוקה: דחוף / בקרוב / לפני יציאה / בטיול
- אחראי, סטטוס, deadline
- בר התקדמות כללי בולט
- צ'קבוקסים שנשמרים בענן

**10. מודול מסמכים** ⭐ Roy ביקש במיוחד
- אחסון מאובטח בענן (Supabase Storage)
- קטגוריות: דרכונים, אישורי טיסה, וויזות, אישורי מלון, ביטוח, רישיון נהיגה
- תיוג לכל משתתף (דרכון של איתמר)
- חיבור לישויות: אישור הזמנה למלון X
- תצוגה: PDF preview, image lightbox
- גישה מהירה מ"Today" - "המסמכים שאצטרך היום"
- אופציה לתיוג "Always Available Offline"

**11. מודול מזג אוויר**
- תחזית 5-7 ימים לכל אזור
- אינטגרציה עם OpenWeather API
- מסך נפרד או widget בכל מודול רלוונטי

**12. שיתוף והתראות בסיסיות**
- הזמנת משתמשים במייל
- Realtime updates (אם Leahy מסמנת משימה, Roy רואה מיד)
- התראות פוש (PWA) למשתמשים מוזמנים

### V2 (אחרי טיול יפן)

**מודול הוצאות**
- רישום הוצאות יומיות
- המרת מטבע אוטומטית (₪/¥/$/€)
- חלוקה בין משתתפים (אם נוסעים עם חברים)
- ייצוא לקובץ אקסל
- גרפים של פילוח לפי קטגוריה

**מודול יומן**
- ערך יומי עם טקסט + תמונות + מיקום
- "היינו פה" - סימון על מפה אוטומטי
- ייצוא כספר זיכרונות PDF
- שיתוף ערכים נבחרים פומבית

**מודול תמונות**
- העלאת תמונות לאלבום הטיול
- שיוך אוטומטי לימים ולמקומות (EXIF)
- שיתוף עם משתתפי הטיול
- אינטגרציה עם יומן

**התראות חכמות**
- "Time to leave" - לפני טיסה
- "Check-out tomorrow"
- "Restaurant booking in 1 hour"

### V3 (לקראת הטיול השני)

**ספרייה ותבניות**
- "טיולים שהיו" - ארכיון מלא
- "תובנות" - מה עבד, מה לא, תקציב בפועל
- "תבניות" - שכפול מבנה לטיול חדש
- "המומלצים שלי" - מלונות, מסעדות, חוויות מטיולים קודמים

**אינטגרציות**
- ייבוא מטיולים קודמים בקליק
- חיפוש "מלונות שאהבתי" כשמחפשים לטיול חדש
- המלצות פנימיות ("בטיול לפורטוגל אהבתם X, אולי גם תאהבו Y?")

## עקרונות מוצר

### עקרונות יסוד

1. **תכנון לתכנון מובייל וגם דסקטופ** - שניהם first-class
2. **המידע שלך, תמיד נגיש** - גישה offline למידע קריטי
3. **שיתוף קל ובטוח** - הזמנת אנשים פשוטה, הרשאות ברורות
4. **מודולרי לפי הצורך** - לא לעמוס במה שלא רלוונטי
5. **יפה, לא מעמיס** - עיצוב נקי, מודרני, נעים לעין

### עקרונות UX

- **3 לחיצות מקסימום** לכל פעולה נפוצה
- **הצג סטטוס תמיד** - לא להשאיר את המשתמש לתהות
- **שמור אוטומטית** - לא לדרוש שמירה ידנית
- **טעינה מהירה** - מתחת ל-2 שניות לכל מסך
- **שגיאות ידידותיות** - תגיד מה לעשות, לא רק מה השתבש

### עקרונות תוכן

- **עברית טבעית** - לא תרגום מאנגלית
- **ללא em dashes** (העדפת Roy)
- **תאריכים בפורמט עברי:** "ה' 23.7"
- **תאריכי לילות:** תמיד צ'ק-אין → צ'ק-אאוט
- **מינימום אמוג'י** - רק כשמוסיף ערך

## נקודות חשובות לפיתוח

### Multi-tenancy מההתחלה
- כל טבלה ב-DB מכילה `user_id` או `trip_id`
- RLS ב-Supabase מבטיח שמשתמשים רואים רק את שלהם
- הזמנות לטיולים דרך טבלת `trip_members`

### גמישות מודולים
- טבלת `modules` שמגדירה אילו מודולים זמינים בפלטפורמה
- טבלת `trip_modules` שמקשרת איזה מודולים פעילים בכל טיול
- כל מודול הוא directory עצמאי תחת `/modules/[module-name]/`
- הוספת מודול חדש לא דורשת שינוי בשאר הקוד

### Theme System
- לבנדר הוא הצבע הברירת מחדל של Tal Travel
- כל טיול יכול לקבל theme accent משלו (לדוגמה: יפן = וורמיליון נוסף)
- כל הצבעים דרך CSS variables כדי לאפשר שינוי dynamic

### Offline-First Critical Data
- Service Worker מקש כל המסמכים, פרטי מלונות, daily plan
- Indicator ברור כשעובדים offline
- סנכרון אוטומטי כשחוזרים online

## מה Tal Travel **לא**

חשוב להבהיר מה לא נכלל ב-MVP ולא בכלל ב-V2-V3:

- **לא** סוכנות נסיעות - אנחנו לא מוכרים טיסות/מלונות
- **לא** מנוע השוואה - אנחנו לא משווים מחירים
- **לא** ספר ביקורות - אין דירוגים פומביים
- **לא** רשת חברתית - אין חברים, לייקים, פוסטים פומביים
- **לא** AI agent שמתכנן לבד - הפלטפורמה היא כלי ידני (גם אם נוסיף AI features עתידיים)

זה כלי אישי לאנשים שאוהבים לתכנן, לא תחליף לתכנון.
